
0) Orden sugerido para la ejecución de módulos
----------------------------------------------

Este es un orden recomendado en que se deberían ejecutar los módulos de Python para crear un ejercicio de programación con variantes a ser usados en EraiEP, partiendo de una cierta estructura de directorio de ejercicio:
1) PythonInputInstrToFunctConverter: python_input_instr_to_funct_converter.py : 
NOTA: Este es un programa principal que usa tres módulos adicionales, así que este es un subsistema compuesto de cuatro módulos.
Programa que convierte las instrucciones de lectura de los códigos base (CB) y las variantes de subcódigo (VS) de un ejercicio en funciones generadoras, según lo indique el usuario.
Este programa usa el programa python_input_instr_identifier.py para encontrar instrucciones de lectura en los CB y VS, y para obtener los valores asociados a los tipos de dato permitidos en una instrucción de lectura (para fines de validación).
Y también usa el módulo input_data_generators.py, que es la librería de funciones generadoras, para poder obtener las referencias a las funciones generadoras existentes que se usan para reemplazar las instrucciones de lectura.

1a) python_input_instr_identifier.py: Programa que se usa para encontrar las instrucciones de lectura en un código de Python.
Este programa usa el módulo que contiene la clase PythonFileInputInstruction.
1aa) PythonFileInputInstruction.py: Contiene la clase PythonFileInputInstruction, que tiene los atributos y métodos relevantes para almacenar y obtener información de las instrucciones de lectura que se encuentren en códigos de Python.

1b) input_data_generators.py: Módulo de Python que contiene la librería de funciones generadoras (y clases) que se pueden usar para convertir instrucciones de lectura en funciones.

----

2) CodeAssembler: assembler.py: Programa ensamblador de códigos base y variantes de subcódigo.
NOTA: Este es un programa principal que usa un módulo adicional, así que este es un subsistema compuesto de dos módulos.
Este programa usa el siguiente módulo:
print_combinations_list.py: Programa que retorna una lista de combinaciones posibles de elementos, dada una lista con el número de elementos a elegir en cada posición o índice de la lista (de los cuales sólo se puede seleccionar uno).

----

3) Programa 1 >> Nota: Este programa también se puede ejecutar, quizás, antes de python_input_instr_to_funct_converter.py o assembler.py. >> DDFilesGenerator: dd_files_generator.py: Programa en el que el usuario define los niveles de dificultad de los CB y VE, y los descriptores de las VE, y genere archivos asociados a ello.

----

4) Programa 2 >> test_case_builders_generator.py, que hace obsoleto a test_case_builders_generator_v0.py, y añade código adicional al CB o VE para producir después archivos de entrada (AE) de casos de prueba (CP), junto con los archivos de salida (AS) de CP, cuando se ejecuta este CB o VE.

----

5) Programa 3 >> test_case_generator.py, que hace obsoleto a test_case_generator_v0.py, y permite generar N casos de prueba (CP) para el ejercicio, con base en el código modificado de CB o VE para generar archivos de CP cuando se ejecute este código N veces, 1 AE y 1 AS de CP por ejecución.

DEPENDENCIAS:
a) Este módulo requiere del módulo de Python llamado shutil.
https://docs.python.org/es/3/library/shutil.html
El módulo shutil ofrece varias operaciones de alto nivel en archivos y colecciones de archivos. En particular, provee funciones que dan soporte a la copia y remoción de archivos. Para operaciones en archivos individuales, véase también el módulo os.
Este se usa para eliminar directorios y copiar archivos.

b) python_input_instr_identifier.py
Este módulo alberga los posibles tipos de datos de las instrucciones de lectura, por medio de los cuales se generan las opciones de FG disponibles al usuario, dependiendo del tipo de instrucción de lectura de datos.
Módulo para obtener las IL de cada archivo, tanto del CB como de las VS.
- Existen también algunas operaciones con archivos directorios que se realizan con este módulo.
- También tiene soporte para verificar si una línea de texto termina con un carácter de nueva línea (newline) - Esto es importante en el tratamiento de archivos .py para obtener, en este caso, las instrucciones de lectura del código fuente.
Este módulo depende, a su vez, de:
b1) PythonFileInputInstruction.py: Clase que representa una IL: Tipo, línea, representación en cadena de caracteres, nombre de variable.

c) input_data_generators.py
Este módulo alberga las funciones generadoras de valores. Los valores generados pueden ser, en términos generales, por cada llamado de función, fijos o aleatorios, un patrón de valores fijos o aleatorios, o una combinación entre estos. Es importante mencionar que este módulo no necesita estar en el mismo directorio que el código fuente que llama a las funciones de este módulo.

4.1) ¿Qué datos necesita el sistema para ser usado?
---------------------------------------------------

Directorio inicial requerido para usar el módulo python_input_instr_to_funct_converter.py:
- Archivo Python del código principal o código base (CB) (main.py)
- Subdirectorios (s1, s2, etc.) en donde cada uno tiene archivos Python asociados a variantes de subcódigo (VS) para el código base (CB) (v1.py, v2.py, etc.) - Cada subdirectorio corresponde a un conjunto de variantes de subcódigo
Tanto main.py como los archivos v1.py, v2.py, etc. conforman una propuesta de solución a un ejercicio de programación en Python con variantes.

NOTA:
En los archivos de CB y de VS, se debe cuidar que la etiqueta #pycass sN esté escrita correctamente, donde N es el número del subcódigo, empezando desde 1.
Ejemplo de uso incorrecto:
# pycass s1
Ejemplo de uso correcto:
#pycass s1
Además, se debe cuidar la sangría (indentation) en donde se ubica la etiqueta, porque todo el código que se ensambla aquí es ubicado a partir de esta sangría.

Note que para el primer, segundo, tercer, cuarto, quinto y sexto módulo NO se necesita el archivo de texto de descripción en el directorio de ejercicios, sino sólo los códigos fuente relevantes. Esto significa que el archivo de texto de descripción del ejercicio sólo es relevante en el SEPPI. Sin embargo, en el diseño propuesto es útil definir de antemano cuáles son los descriptores que se usarán, puesto que el sistema semiautomático de generación de ejercicios es capaz de generar los archivos relacionados con esos descriptores, y por ende, puede servir como ayuda para ello.

Para el segundo módulo, se necesita el directorio con los archivos modificados por el primer módulo.

Para el tercer módulo, se necesita el directorio con los archivos generados por el segundo módulo. En este momento ya se deben haber establecido las variantes de ejercicio (VE) en el directorio principal de los ejercicios a través de los archivos resN.py.

IMPORTANTE: Antes de usar el tercer módulo, se debería saber cuántos descriptores están asociados a cada subcódigo.
También, aunque no se necesite incluir el archivo de texto de descripción (descripcion.txt) en el directorio principal de los ejercicios para que el módulo 3 funcione, el docente debería saber de antemano cuál es la descripción general, y cuáles son las partes del texto de la descripción del ejercicio que deberían cambiar dependiendo de las variantes de los subcódigos del ejercicio, y de manera más general, de las variantes de ejercicio.

Para el cuarto módulo, se necesita el directorio con los archivos generados por el tercer módulo.

Para el quinto módulo, se necesita el directorio con los archivos generados por el cuarto módulo.

IMPORTANTE: Antes de usar el quinto módulo, se debería saber cuántos casos de prueba, en total, se quieren generar para cada variante de ejercicio.

Para el sexto módulo, se necesita el directorio con los directorios generados por el quinto módulo. En este sexto módulo, se hará uso de los archivos de dificultadN.txt y descriptoresN.txt creados en el tercer módulo, así como los directorios específicos a variantes de ejercicio (varN). En los casos anteriores, N es el número de la variante de ejercicio.

ADVERTENCIA [REQUIERE CONFIRMACIÓN]: Es posible que para el uso de los archivos del ejercicio en SEPPI sea encesario cambiar el nombre del directorio principal, para que este se llame ejercicioM, siendo M el nombre del grupo de ejercicios.
